How it works

Three steps, no scripting required

AutoReview runs as a background accessibility service. Once it's granted permission, it watches the screen you're on and acts only when it finds what it's looking for.

01

Grant accessibility access

Turn on the AutoReview service in Android's Accessibility settings. This is what lets it read the on-screen UI tree — nothing more.

02

Open the target app

Launch the app you want to automate. AutoReview stays idle in the background until it detects a matching screen.

03

It scans, matches, taps

The service walks the node tree, fuzzy-matches the target label, and dispatches the tap — even if the layout shifts between versions.

Built for reliability

Coordinates break. Node trees don't.

Most "auto-tap" tools record x/y positions and replay them blind. AutoReview reads the actual accessibility tree, so it keeps working across resolutions, densities, and layout changes.

Fuzzy text matching

Finds the right control even when labels vary slightly across app updates or locales.

Node-tree traversal

Walks AccessibilityNodeInfo directly — never taps a blind coordinate.

Runs quietly in the background

A lightweight foreground-safe service that only steps in when it finds a match on screen.

Sideload-ready signing

Every release is signed and shipped through GitHub Releases — install directly, no store review wait.

Nothing leaves the device

No screen recording, no network calls to a server, no data collection. The tree stays local.

Retries, not failures

Missed a match on a slow-loading screen? It re-scans instead of giving up.
